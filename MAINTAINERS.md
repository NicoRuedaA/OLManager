## Maintainer responsibilities

- Review new issues and apply `status:needs-review` or `status:approved`.
- Ensure PRs link approved issues and carry exactly one `type:*` label.
- Protect `main` and `development` from direct pushes.
- Own release PRs from `development` to `main`.
- Keep GPL lineage, third-party attribution, and data provenance records current.
- Decide when signing/notarization secrets are available for packaged releases.

## Contact placeholders

The public maintainer and security contact must be replaced before the repository is widely announced. Until then, contributors should use GitHub profile contact methods for the repository owner and avoid posting vulnerabilities publicly.
